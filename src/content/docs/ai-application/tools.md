Tool 是提供给大模型的**业务能力说明书**。它告诉模型这项能力叫什么、能解决什么问题、需要哪些参数。

## RPC 与 Tool 的关系

Feign RPC 接口负责真正执行查询或操作；Tool 则是面向大模型的语义契约。

```text
Tool 定义
├── name：稳定、清晰的能力名称
├── description：什么情况下应该使用
├── input schema：模型必须提供的参数
└── executor：调用 Feign 或其他业务服务
```

一个 RPC 接口可以一对一包装为 Tool，但更合理的设计通常是面向业务任务。例如“查询维修发货状态”这个 Tool，内部可能组合维修单、物流和用户权限等多个服务。

## 一次典型调用

1. AI 应用把用户问题和可用 Tool 定义发给模型。
2. 模型决定是否调用 Tool，并返回名称和结构化参数。
3. AI 应用校验参数和权限，再调用业务服务。
4. AI 应用把查询结果交回模型，由模型组织面向用户的回答。

模型只负责选择和生成参数，真正的权限、安全与业务执行仍然由应用服务控制。
